Module Interaction Map

How the orchestrator's files call each other, in the order they fire, for a single run. Solid arrows are direct calls carrying data. Dashed arrows are validation/logging — no control passes, only a check or a record. intake.py is built and tested today; everything else below is the planned shape it will call into next.

Two files are deliberately never in the "calls into" column of anything upstream: kernel.py and calibration.py. They're imported, not orchestrated — that asymmetry is the point. A file that no one calls into can't be talked out of its job by anything happening in the loop above it.
