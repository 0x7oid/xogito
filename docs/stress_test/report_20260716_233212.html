Single-threaded it processes 3,000 images per hour and we need 12,000 per hour. Dana also claims there is zero documented evidence that the GIL limits threaded CPU-bound performance in CPython. The result

The optimal model for this service is multiprocessing, as it is the only option that allows for true parallel execution of CPU-bound tasks. Dana's claim of an 8x speedup via threading is theoretically impossible, and the provided findings confirm that the global interpreter lock prevents parallel thread execution.

What happened, in short

Dana's claim about a significant speed boost from using threads was proven incorrect based on these findings.
