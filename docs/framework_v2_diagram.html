REASONING FRAMEWORK v2 — patched architecture

White boxes = original pipeline  ·  orange = the nine fixes  ·  dashed purple = Protected Kernel (code-enforced, not prompt-enforced)

FIX INDEX — your issue list → the mechanism on the diagram

  1. FIX 1 — Confidence system: evidence ladder becomes typed; every position carries its justification type, so incommensurable evidence is never compared on one scale.
  2. FIX 2 — Weakest-link reasoning: replaced by support aggregation — independent lines of medium evidence can promote a claim; correlated sources are clustered and counted once.
  3. FIX 3 — Anchor conflict: anchors are never overturned, but evidence against them raises an anchor tension flag that survives all the way into the Final Report.
  4. FIX 4 — Disagreement / branching: Contested claims hit a decision function — ignore (no fan-out), collapse (probe shows same conclusion), or bounded branch with labeled conditional conclusions.
  5. FIX 5 — Evaluator god: verdicts are proposals; a code-level harness checks evidence IDs, blind-re-reads evidence, dual-passes high-stakes claims, applies a promotion ratchet, and logs calibration.
  6. FIX 6 — Problem evolvement: the system may propose revised success criteria at a Checkpoint; the user ratifies through the same gate pattern; revision distance is capped.
  7. FIX 7 — Summary prose: the Compressor carries structured state verbatim as data and only summarizes narrative; a Python assert guarantees no Contested flag disappears.
  8. FIX 8 — Verification recursion: verification artifacts never spawn verification tasks; the depth cap forces Contested rather than looping.
  9. FIX 9 — Parallelism: executors read versioned snapshots; if a premise is demoted after the snapshot, the dependent artifact is auto-queued for re-evaluation (or evaluate serially in v1).
  10. KERNEL — the invariants sit above user policy and are enforced by the orchestrator’s Python control flow, not by prompting.
